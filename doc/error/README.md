# 错误记录说明

本目录用于记录开发过程中遇到的错误，便于后续避免重复问题。

## 文件命名规范

```
YYYY-MM-DD_错误简述.md
```

例如：
- `2025-01-29_ES模块导入错误.md`
- `2025-01-29_Socket事件名拼写错误.md`

## 错误文档模板

```markdown
# 错误标题

**日期**: YYYY-MM-DD
**模块**: 涉及的模块名
**严重程度**: 高/中/低

## 错误描述

简要描述错误现象

## 错误原因

分析错误产生的原因

## 解决方案

如何修复这个错误

## 预防措施

如何避免再次发生
```

## 使用流程

1. **写代码前** - 先读取本目录下的所有错误记录
2. **遇到错误时** - 立即创建错误记录文档
3. **代码审查时** - 检查是否存在已记录的错误模式
