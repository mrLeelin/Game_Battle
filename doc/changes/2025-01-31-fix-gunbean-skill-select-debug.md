# 枪豆人肉鸽技能无效调试日志

## 问题描述
肉鸽升级后选择的技能完全没有效果，无法定位问题原因。

## 根本原因
服务端 `handleSkillSelect` 方法中多个 `return` 语句没有日志输出，导致技能选择失败时静默失败，无法诊断问题。

## 修复方案
在 `handleSkillSelect` 方法中添加详细的调试日志，定位技能选择失败的具体原因。

## 修改内容
- 文件：`server/games/gunbean/GunBeanHandler.js`
- 行数：343-372

添加的日志点：
1. 收到技能选择事件时打印完整的请求数据
2. 玩家不存在或没有房间时输出错误
3. 游戏不存在时输出错误
4. 游戏玩家不存在时输出错误（包含游戏中所有玩家列表）
5. 玩家选择技能时打印技能ID
6. 技能不存在时输出错误

## 测试验证
1. 启动游戏并升级
2. 选择技能
3. 查看服务端控制台日志
4. 根据日志定位具体问题

## 可能的问题原因
- `gamePlayer` 获取失败（socket.id 不匹配）
- `skill` 获取失败（skillId 不正确）
- 游戏状态不正确（game 未找到）
